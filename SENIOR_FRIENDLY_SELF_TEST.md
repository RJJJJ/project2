# Senior-friendly Self Test

目標：驗證 40+ / 非技術用戶是否能在 30 秒內理解並使用 Simple Mode。

## Checklist

評分欄位：PASS / FAIL / NOTES。

| # | 檢查項目 | PASS / FAIL / NOTES |
|---|---|---|
| 1 | 手機寬度是否可讀 | |
| 2 | 30 秒內是否知道第一步要選地區 | |
| 3 | 30 秒內是否知道第二步要輸入購物清單 | |
| 4 | 是否能找到「幫我找最抵買法」按鈕 | |
| 5 | 第一眼是否看到推薦超市 | |
| 6 | 第一眼是否看到預計總價 | |
| 7 | 商品明細是否沒有 product_oid / plan_type / ranking_factors | |
| 8 | 錯誤訊息是否像人話 | |
| 9 | 是否不需要理解「API / provider / SQLite / JSONL」 | |
| 10 | 是否知道資料範圍約 500 米 | |
| 11 | 是否知道結果只供參考，實際店內價格可能變 | |
| 12 | 是否能在 1 分鐘內完成一次查詢 | |

12 項至少 9 項 PASS 才算可展示。

## 自測案例

### Case 1
- 地區：高士德
- 輸入：一包米、兩支洗頭水、一包紙巾

### Case 2
- 地區：花城
- 輸入：一盒牛奶、一支食油、一包米

### Case 3
- 地區：關閘 / 台山 / 筷子基（視 config 現有 point 而定）
- 輸入：牙膏、洗衣液、紙巾

## 真實用戶測試指引

- 找 1 個非技術用戶。
- 不講解，只讓他自己操作。
- 記錄卡住位置。
- 不問「好不好看」，只看能否完成任務。
- 記錄最大 1 個 UX 問題。

## Real-user query robustness notes

- 真實用戶可能輸入空格分隔清單，例如「兩包麵 一包薯條 四包薯片 油 糖 M&M」。
- 找不到全部商品不是系統錯誤；系統會先顯示 partial result / 已找到商品。
- 建議用戶輸入更具體名稱，例如「食油」、「薯片」、「白米」。
